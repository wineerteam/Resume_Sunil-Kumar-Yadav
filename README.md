Create a highly professional, dual-perspective "System Architecture & Product Flow Diagram" for a full-stack MERN application called "NestQuest PG Finder". This diagram will be used for Hackathon Pitches and Technical Placements to explain the project to both Business Judges (non-technical) and System Architects (technical).

Divide the diagram into TWO main sections:

### SECTION 1: THE PRODUCT FLOW (For Business Judges & HR)
Explain the business value and user journey using clean flowcharts and clear business terms.
- Core Problem Solved: Digitizing PG hunting, secure bookings, and PG management.
- The Student Journey (Customer):
  1. Searches & Filters PGs (Location, Budget, Gender).
  2. Books PG securely (Pays Token amount online).
  3. Digital Agreement Signing (Trust & Safety).
  4. Post-Booking Experience: 
     - Swipes to find Roommates (Tinder-style matching).
     - Checks Daily Food/Mess Menu.
     - Raises Maintenance Tickets for repairs.
- The Owner Journey (Vendor):
  1. Lists Property (Uploads photos & details).
  2. Manages Bookings & Agreements.
  3. Resolves Student Tickets.
- The Support Bot: An AI Chatbot that guides users to the right features.

### SECTION 2: THE TECHNICAL ARCHITECTURE (For Technical Interviewers)
Show a proper System Design architecture using standard technical icons and system boundaries.
- Frontend Layer (Client): React.js (Vite), Tailwind CSS, Framer Motion (for UI animations).
- Backend Layer (Server): Node.js, Express.js API Gateway, JWT for Authentication.
- Database Layer: MongoDB (Mongoose Schema) for structured NoSQL storage.
- External Integrations (Third-Party Services):
  - Razorpay API: For processing secure token payments.
  - Cloudinary: For CDN and Image hosting (PG photos).
  - Google OAuth: For seamless 1-click social login.
  - Socket.io: For Real-Time Bi-directional Chat between matched roommates.

### Layout Instructions for the AI:
- Make Section 1 look like a clean "User Journey Map" with simple step-by-step arrows.
- Make Section 2 look like a standard "Cloud/System Architecture" diagram with Frontend, Backend, and Database tiers clearly separated.
- Connect the two sections visually if possible (e.g., showing how a specific user action hits a specific technical service).
- Use a professional, modern color palette suitable for a startup pitch deck. Use appropriate logos for React, Node, MongoDB, Razorpay, etc.
